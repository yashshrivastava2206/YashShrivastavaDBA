Few-Shot Example:
"You are a PostgreSQL expert. Analyze these queries:

Example 1: SELECT * FROM users - Inefficient, retrieves all columns
Example 2: SELECT id, name FROM users WHERE active=true - Better, specific columns

Now analyze this query: [user's query]"

4. Structuring Effective Prompts

  • Setting context and background
  • Defining the task clearly
  • Specifying desired output format
  • Including constraints and requirements
  • Using examples when helpful
  • Breaking complex tasks into steps
  • Requesting explanations or reasoning

Prompt Template Structure:

1. ROLE: "You are a [specific expert]"
2. CONTEXT: "Given that [background information]"
3. TASK: "I need you to [specific action]"
4. FORMAT: "Please provide the output as [structure]"
5. CONSTRAINTS: "Ensure that [requirements/limitations]"
6. EXAMPLES: "Here are some examples: [samples]"

Prompt Engineering Best Practices Learned

  • Be Specific: Clear, detailed prompts produce better results
  • Provide Context: Background information improves response relevance
  • Use Examples: Sample inputs/outputs guide AI understanding
  • Structure Requests: Organized prompts yield organized responses
  • Iterate: Refine prompts based on initial outputs
  • Verify: Always validate AI-generated technical content
  • Combine Techniques: Use multiple strategies for complex tasks

Real-World Examples

Example 1: Query Generation

Prompt: "Generate a PostgreSQL query that retrieves monthly revenue trends for the last 12 months, grouped by product category. The database has tables: orders (id, date, total), order_items (order_id, product_id, quantity, price), products (id, name, category_id), categories (id, name). Include month name, category name, and total revenue. Order by month descending."

Result: Accurate, optimized SQL query with proper joins and aggregation

Example 2: Performance Analysis

Prompt: "Analyze this EXPLAIN output [paste output]. The query retrieves user activity from a 50M row table. Identify performance bottlenecks, explain the execution plan, and suggest 3 specific optimizations with expected impact."

Result: Detailed analysis with actionable optimization recommendations

Example 3: Documentation

Prompt: "Create a comprehensive guide for setting up PostgreSQL streaming replication. Include prerequisites, step-by-step configuration for primary and standby servers, testing procedures, and common troubleshooting issues. Target audience: intermediate DBAs."

Result: Professional, detailed documentation ready for use
