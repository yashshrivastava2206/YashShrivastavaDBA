Project Overview

The Easy Fitness Club Manager is an intuitive desktop software solution designed to streamline and simplify fitness club operations. Built using Java and MySQL, this comprehensive management system provides secure data storage, efficient member management, payment tracking, and reporting capabilities tailored specifically for fitness clubs, gyms, and wellness centers.

The application features a user-friendly interface that allows fitness club staff to manage all aspects of their operations from a single platform. With robust security features, automated calculations, and real-time reporting, the system significantly improves operational efficiency and reduces manual workload for fitness club administrators.

Problem Statement

Fitness clubs and gyms face several operational challenges:

  • Manual member registration and record-keeping prone to errors
  • Difficulty tracking membership payments and outstanding dues
  • Lack of centralized system for managing member information
  • Time-consuming manual calculations for fees and membership periods
  • Security concerns with paper-based member data storage
  • Inefficient reporting and analytics for business decisions
  • Poor tracking of membership expiration dates
  • Difficulty managing multiple membership types and pricing
Need for Automation

Fitness club owners needed an affordable, easy-to-use system that could replace manual processes, improve data security, and provide better visibility into their business operations without requiring extensive technical expertise.

Database Design

Normalized database schema with the following key tables:

  • Members: Personal information, contact details, photo, join date
  • Memberships: Plan types, durations, pricing, status
  • Payments: Payment records, amounts, dates, receipt numbers
  • Users: System users with role-based access control
  • Attendance: Member check-in/check-out logs
-- Sample database schema CREATE TABLE members ( member_id INT PRIMARY KEY AUTO_INCREMENT, first_name VARCHAR(50) NOT NULL, last_name VARCHAR(50) NOT NULL, phone VARCHAR(15) UNIQUE, email VARCHAR(100), address TEXT, join_date DATE, photo BLOB, status ENUM('Active', 'Inactive', 'Expired') ); CREATE TABLE memberships ( membership_id INT PRIMARY KEY AUTO_INCREMENT, member_id INT, plan_type VARCHAR(50), start_date DATE, end_date DATE, amount DECIMAL(10,2), FOREIGN KEY (member_id) REFERENCES members(member_id) ); CREATE TABLE payments ( payment_id INT PRIMARY KEY AUTO_INCREMENT, member_id INT, amount DECIMAL(10,2), payment_date DATE, receipt_number VARCHAR(20) UNIQUE, payment_method VARCHAR(20), FOREIGN KEY (member_id) REFERENCES members(member_id) );

Technical Implementation

Object-Oriented Design

  • Model-View-Controller (MVC) architecture pattern
  • Separate classes for Member, Membership, Payment, User entities
  • Database access layer with connection pooling
  • Service layer for business logic
  • Reusable UI components and custom widgets

Security Features

  • Password hashing using MD5/SHA for secure storage
  • SQL injection prevention using prepared statements
  • Role-based access control (Admin, Staff, Viewer)
  • Session management and automatic logout
  • Audit logging for sensitive operations

Data Validation

  • Input validation for all form fields
  • Phone number and email format validation
  • Date range validation for memberships
  • Payment amount validation and calculation checks
  • Duplicate entry prevention (phone, email)
