Project Overview

The DBA Task Automation Framework is a comprehensive enterprise solution designed to automate routine database administration tasks across 40+ production PostgreSQL servers. This framework integrates multiple automation tools, monitoring systems, and reporting mechanisms to create a centralized, self-managing database infrastructure.

Built using a combination of Shell scripting, Java, Cron scheduling, and Ansible configuration management, the framework handles everything from automated backups and health checks to performance monitoring and intelligent alerting. The system has successfully reduced manual DBA effort by 70% while improving database reliability and performance visibility.

Key Achievement

Transformed manual, time-consuming DBA processes into an automated, intelligent system that monitors, reports, and maintains 40+ database servers with minimal human intervention, allowing the DBA team to focus on strategic improvements and optimization.

Business Challenge

Managing 40+ production PostgreSQL servers manually was becoming increasingly challenging and resource-intensive. Key problems included:

  • Time-Consuming Manual Tasks: DBAs spent 60-70% of their time on routine maintenance, backups, and monitoring
  • Delayed Problem Detection: Issues were discovered reactively, often after impacting users
  • Inconsistent Monitoring: Manual checks were inconsistent across servers and prone to human error
  • Limited Visibility: No centralized view of database health and performance across all servers
  • Reactive Management: Team was constantly firefighting instead of proactive optimization
  • Report Generation Overhead: Manual report generation consumed significant time
  • Communication Gaps: Stakeholders lacked timely information about database status

1. pg_profile Automation Script


#!/bin/bash
# Automated pg_profile Report Generation
# Runs twice daily via cron: 0 9,18 * * *

SERVERS_LIST="/etc/dba-automation/servers.txt"
REPORT_DIR="/var/reports/pg_profile"
CENTRAL_SERVER="monitoring-server.company.com"
DATE=$(date +%Y%m%d_%H%M)
MAIL_LIST="dba-team@company.com"

# Create report directory
mkdir -p $REPORT_DIR/$DATE

# Function to generate pg_profile report
generate_report() {
local server=$1
local db=$2

echo "Generating pg_profile for $server/$db..."

psql -h $server -U monitor -d $db -c "
SELECT profile_create_report(
(SELECT max(server_id) FROM servers WHERE server_name='$server'),
(SELECT max(sample_id)-48 FROM samples),
(SELECT max(sample_id) FROM samples),
'daily_report'
);" > $REPORT_DIR/$DATE/${server}_${db}_profile.html

if [ $? -eq 0 ]; then
echo "✓ Report generated: ${server}_${db}"
return 0
else
echo "✗ Failed: ${server}_${db}"
return 1
fi
}

# Generate reports for all servers
while IFS='|' read -r server database; do
generate_report "$server" "$database" &
done < $SERVERS_LIST

wait

# Transfer to central server
rsync -avz $REPORT_DIR/$DATE/ \
$CENTRAL_SERVER:/var/www/html/reports/pg_profile/$DATE/

# Generate summary email
generate_summary() {
echo "pg_profile Report Summary - $(date)"
echo "=================================="
echo "Total Servers: $(wc -l < $SERVERS_LIST)"
echo "Reports Generated: $(ls $REPORT_DIR/$DATE/*.html 2>/dev/null | wc -l)"
echo "Failed Reports: $(grep -l "✗ Failed" /var/log/pg_profile.log | wc -l)"
echo ""
echo "View reports at: http://$CENTRAL_SERVER/reports/pg_profile/$DATE/"
}

# Send email via Zimbra
generate_summary | mail -s "pg_profile Daily Report - $DATE" $MAIL_LIST

# Cleanup old reports (keep 30 days)
find $REPORT_DIR -type d -mtime +30 -exec rm -rf {} \;

2. pgBadger Hourly Update Script


#!/bin/bash
# pgBadger Hourly Log Analysis
# Runs every hour via cron: 0 * * * *

SERVERS_LIST="/etc/dba-automation/servers.txt"
LOG_DIR="/var/lib/postgresql/logs"
REPORT_DIR="/var/www/html/pgbadger"
TEMP_DIR="/tmp/pgbadger"

mkdir -p $TEMP_DIR

# Function to process logs
process_server_logs() {
local server=$1
local server_clean=$(echo $server | tr '.' '_')

echo "Processing logs for $server..."

# Fetch last hour's logs
ssh postgres@$server "cat $LOG_DIR/postgresql-*.log | \
awk -v d=\"\$(date -d '1 hour ago' '+%Y-%m-%d %H')\" '\$0 >= d'" \
> $TEMP_DIR/${server_clean}_latest.log

# Generate pgBadger report
pgbadger --prefix '%t [%p]: [%l-1] user=%u,db=%d,app=%a,client=%h ' \
--outfile $REPORT_DIR/${server_clean}.html \
--incremental \
--retention 7 \
$TEMP_DIR/${server_clean}_latest.log

# Extract top slow queries
grep -A 5 "Slowest queries" $REPORT_DIR/${server_clean}.html | \
grep -E "duration:|query:" > $TEMP_DIR/${server_clean}_slow.txt

# Alert if critical slow queries found
if grep -q "duration: [5-9][0-9]\{3\}" $TEMP_DIR/${server_clean}_slow.txt; then
echo "ALERT: Critical slow queries on $server" | \
mail -s "Slow Query Alert - $server" dba-team@company.com
fi
}

# Process all servers in parallel
while read -r server; do
process_server_logs "$server" &
done < $SERVERS_LIST

wait

# Update index page
generate_index_html

# Cleanup temp files
rm -rf $TEMP_DIR/*

3. Health Check & Alerting Script


#!/bin/bash
# Database Health Check & Alerting
# Runs every 5 minutes via cron

check_database_health() {
local server=$1
local alerts=()

# Check connections
local conn_count=$(psql -h $server -U monitor -t -c \
"SELECT count(*) FROM pg_stat_activity;")
local max_conn=$(psql -h $server -U monitor -t -c \
"SHOW max_connections;" | xargs)
local conn_pct=$((conn_count * 100 / max_conn))

if [ $conn_pct -gt 80 ]; then
alerts+=("High connection usage: ${conn_pct}%")
fi

# Check replication lag
local lag=$(psql -h $server -U monitor -t -c \
"SELECT EXTRACT(EPOCH FROM (now() - pg_last_xact_replay_timestamp()));" 2>/dev/null)

if [ ! -z "$lag" ] && [ $(echo "$lag > 60" | bc) -eq 1 ]; then
alerts+=("Replication lag: ${lag}s")
fi

# Check disk space
local disk_usage=$(ssh $server "df -h /var/lib/postgresql | tail -1 | awk '{print \$5}' | sed 's/%//'")

if [ $disk_usage -gt 85 ]; then
alerts+=("High disk usage: ${disk_usage}%")
fi

# Check for long-running queries
local long_queries=$(psql -h $server -U monitor -t -c \
"SELECT count(*) FROM pg_stat_activity
WHERE state = 'active'
AND now() - query_start > interval '10 minutes';")

if [ $long_queries -gt 0 ]; then
alerts+=("Long-running queries: ${long_queries}")
fi

# Send alerts if any issues found
if [ ${#alerts[@]} -gt 0 ]; then
{
echo "Database Health Alert - $server"
echo "================================"
echo "Timestamp: $(date)"
echo ""
for alert in "${alerts[@]}"; do
echo "⚠ $alert"
done
} | mail -s "Health Alert - $server" dba-team@company.com
fi
}

# Run health checks for all servers
while read -r server; do
check_database_health "$server"
done < /etc/dba-automation/servers.txt

4. Ansible Playbook for Configuration


---
# Ansible playbook for DBA automation deployment
- name: Deploy DBA Automation Framework
hosts: postgresql_servers
become: yes

vars:
automation_dir: /opt/dba-automation
scripts_dir: "{{ automation_dir }}/scripts"
log_dir: /var/log/dba-automation

tasks:
- name: Create automation directories
file:
path: "{{ item }}"
state: directory
owner: postgres
group: postgres
mode: '0755'
loop:
- "{{ automation_dir }}"
- "{{ scripts_dir }}"
- "{{ log_dir }}"

- name: Copy automation scripts
copy:
src: "{{ item }}"
dest: "{{ scripts_dir }}/"
owner: postgres
group: postgres
mode: '0750'
loop:
- pg_profile_generate.sh
- pgbadger_update.sh
- health_check.sh
- backup_automation.sh

- name: Configure cron jobs
cron:
name: "{{ item.name }}"
minute: "{{ item.minute }}"
hour: "{{ item.hour }}"
job: "{{ scripts_dir }}/{{ item.script }} >> {{ log_dir }}/{{ item.log }} 2>&1"
user: postgres
loop:
- { name: "pg_profile morning", minute: "0", hour: "9",
script: "pg_profile_generate.sh", log: "pg_profile.log" }
- { name: "pg_profile evening", minute: "0", hour: "18",
script: "pg_profile_generate.sh", log: "pg_profile.log" }
- { name: "pgbadger hourly", minute: "0", hour: "*",
script: "pgbadger_update.sh", log: "pgbadger.log" }
- { name: "health check", minute: "*/5", hour: "*",
script: "health_check.sh", log: "health_check.log" }

- name: Install required packages
package:
name:
- postgresql-contrib
- pgbadger
- rsync
state: present

Challenges & Solutions

Challenge 1: Scale - 40+ Servers

Problem: Sequential execution would take too long for timely reports

Solution: Implemented parallel processing using bash background jobs with wait commands, reducing total execution time from 2 hours to 15 minutes

Challenge 2: Network Reliability

Problem: Network issues causing script failures and incomplete reports

Solution: Added retry logic, timeout handling, and failure recovery mechanisms. Implemented circuit breaker pattern for persistent failures

Challenge 3: Report Storage

Problem: Rapid growth of report files consuming disk space

Solution: Implemented intelligent retention policies (hourly: 7 days, daily: 30 days, weekly: 90 days) with automatic compression and archival

Challenge 4: Alert Fatigue

Problem: Too many alerts causing team to ignore notifications

Solution: Implemented intelligent alerting with priority levels, deduplication, and threshold tuning based on historical patterns

Lessons Learned
  • Start Small, Scale Gradually: Began with 5 servers before rolling out to all 40+
  • Robust Error Handling: Network and server issues are inevitable; planning for failures is critical
  • Idempotency Matters: Scripts should be safe to run multiple times without side effects
  • Logging is Essential: Comprehensive logging made debugging and optimization possible
  • Monitoring the Monitors: Built self-health checks for the automation system itself
  • Documentation: Thorough documentation enabled team adoption and maintenance
  • Feedback Loops: Regular team feedback led to continuous improvements
